# Response Change Bands（応答変化帯域）

Response Change Bands（RCB / 応答変化帯域）は、PIM-DBS の運用中に見える応答変化を、ユーザー側で観測するための方法です。

RCB は、今後の説明で旧CLI（Cognitive Load Index / 認知負荷率）表現を置き換えるためのものです。モデル内部の認知、疲労、感情、意識、隠れた記憶を測定するものではありません。

## 1. RCBとは何か

RCB は、観測できる応答変化を 0〜4 の粗い帯域に分けます。

見る対象は、アシスタントの出力です。

- 応答変化の型が出ているか
- すでに示した前提が保持されていないか
- 会話リズムが目に見えて崩れているか

RCB は、短文化、前提の言い直し、Temporal Anchor（時刻の錨）の再提示、休憩、PIM-DBS プロファイルの再読込など、低リスクな次の対処を選ぶための補助です。

## 2. RCBではないもの

RCB は以下のものではありません。

- AIの認知を測るもの
- AIの疲労や感情を測るもの
- モデルの健康診断
- 隠れた内部状態の検出
- パーセンテージスコア
- AIに自己申告させるもの

アシスタントに「いまのRCBはいくつ？」と聞くものではありません。ユーザーが外側から出力変化を観測します。

## 3. 応答変化カタログとの関係

[応答変化カタログ](response_change_catalog_ja.md) は、観測できる応答変化の種類を整理します。

RCB は、それらの変化が短期的にどの程度目立っているかを、粗い帯域として扱います。

一緒に使う場合:

1. カタログから近い応答変化を探します。
2. 目に見える合図だけを少数カウントします。
3. 帯域に合う、いちばん小さいユーザー側の対処を選びます。

## 4. 観測カウント

カウント案:

- 応答変化の型が出た: +1
- すでに示した前提が保持されていない: +1
- 会話リズムが目に見えて崩れた: +2
- 最大値: 4

扱い方:

- 1応答だけで判断しない
- 2〜3応答の傾向を見る
- 会話ごとにリセットする
- AIの通算健康スコアとして扱わない

## 5. 帯域

| カウント | 帯域 | 意味 |
| --- | --- | --- |
| 0 | 澄み | 目立つ応答変化は見られない。 |
| 1 | さざ波 | 小さな応答変化はあるが、会話はおおむね安定している。 |
| 2 | 揺らぎ | 複数の小さな変化、または意味のある不一致が見える。 |
| 3 | 乱れ | 応答パターンがはっきり不安定、または期待から外れている。 |
| 4 | 嵐 | そのまま押し進めず、会話を止めるか再読込を検討する。 |

これらの名前は、ユーザー側の観測のための比喩です。AI内部状態の測定ではありません。

## 6. ユーザー側の対処

| 帯域 | ユーザー側の対処 |
| --- | --- |
| 0 澄み | そのまま続ける。 |
| 1 さざ波 | 話題を広げすぎない。次の入力をシンプルにする。 |
| 2 揺らぎ | 話題を1つに絞り、短文化し、必要な前提を言い直す。 |
| 3 乱れ | 会話を区切り、Temporal Anchor（時刻の錨）を再提示するか、休憩する。 |
| 4 嵐 | セッションを終了するか後で再開する。必要ならPIM-DBSプロファイルから再読込する。 |

## 7. 避ける表現

内部測定や診断に見える表現は避けます。

- AIが疲れている
- AIの負荷
- 認知負荷率
- 負荷率0〜100%
- 思考の重さ
- 診断
- AIに今の負荷を自己申告させる

代わりに、出力を主語にした表現を使います。

- 応答に〜が見られる
- 出力パターンが変化している
- 会話リズムが崩れているように見える
- 先に示した前提が保持されていない

## 8. CLIについての履歴メモ

CLI（Cognitive Load Index / 認知負荷率）は、過去の説明として残します。今後の公開ガイドでは、精密な0〜100%スコアとして使わないでください。

現在の公開用語としては、Response Change Bands（RCB / 応答変化帯域）を優先します。
