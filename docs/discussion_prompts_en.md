# PIM-DBS Discussion Prompts

This document collects questions for external discussion about PIM-DBS and related user-side continuity tools.

It is not a promotional script. It is meant to help readers, researchers, creators, and community members examine what should be preserved, what should remain private, and how reconstruction can be evaluated without claiming persistent AI memory or consciousness.

PIM-DBS uses metaphors such as persona, memory, forge, shield, and ark. In technical discussion, these refer to user-side context profiles and reconstruction workflows, not hidden AI states.

## 1. General Questions

- What should users preserve for long-term AI continuity?
- What should never be preserved in a public template?
- How can restoration be verified without claiming persistent AI memory?
- How can user-side continuity tools avoid over-anthropomorphism?
- What parts of a long-running AI interaction are useful context, and what parts are too private to store?
- How should a profile distinguish user-recorded context, AI-claimed context, and inferred summaries?
- What language helps people understand continuity without implying that the AI has an inner life?

## 2. For Researchers

- How should user-side AI continuity be studied without assuming AI consciousness?
- What are safe evaluation methods for response-style reconstruction?
- How can provenance labels reduce false claims about memory?
- What metrics could evaluate tone, role consistency, boundaries, and user expectations without measuring hidden model states?
- How should dependency risk be studied when continuity tools make AI interactions feel more stable?
- What kinds of longitudinal user research would be ethical for persona continuity tools?
- How should researchers separate metaphorical user language from technical claims?

## 3. For Creators / Writers

- How can fictional AI companions be documented safely?
- What is the difference between a character profile and a private interaction profile?
- How can Dialogue Exemplars be used without exposing private conversations?
- What should creators include in a public character profile, and what should remain in private notes?
- How can creator-authorized profiles preserve style without encouraging unauthorized imitation of real people or protected characters?
- How should fictional canon, allowed interpretation range, and forbidden topics be represented?
- When does a useful character reference become too close to a private voiceprint or unpublished author note?

## 4. For AI Companion / VR / Metaverse Communities

- What safety boundaries are needed when users build long-term relationships with AI characters?
- How should public templates be separated from private instances?
- How can restoration checks reduce dependency or over-identification?
- What should a VR or metaverse AI character disclose about its fictional and generated nature?
- How should creator-managed profiles handle role boundaries, consent, and public performance contexts?
- What kinds of verification probes would help detect character drift in an AI-powered NPC or companion?
- How can communities support continuity while avoiding claims of persistent AI memory or personhood?

## 5. Practical Discussion Starters

These short prompts can be adapted for Reddit, GitHub Discussions, X, note, or community posts.

- I am exploring a user-side continuity format for AI interactions. What do you think should never be included in a public AI persona template?
- How would you verify that an AI character profile reloaded correctly without claiming the AI has persistent memory?
- For long-term AI companion use, what boundaries should be written into the profile instead of left implicit?
- If a fictional AI character has a public profile and a user's private interaction history, where should the line be drawn?
- What provenance labels would help prevent confusion between user records, AI claims, and inferred summaries?
- For VR or metaverse AI characters, what safety checks should exist before a creator-authorized persona is deployed publicly?
- How can a project use metaphors like memory or ark while staying clear that it is not making claims about AI consciousness?
- What would make a persona continuity protocol useful for creators without enabling unauthorized imitation or private data leakage?

## 6. Safety Framing

When using these prompts publicly, keep the framing clear:

- PIM-DBS is a user-side context profile and reconstruction workflow.
- It does not claim AI consciousness, a soul, or hidden persistent memory.
- Public examples should use fully fictional personas only.
- Do not publish raw Dialogue Exemplars from real private conversations.
- Do not use real person voiceprints, real private personas, or sensitive relationship context as public examples.
