# PIM-DBS 公開用 / 私的利用ガイド

PIM-DBS のプロファイルには、個人的な文脈が含まれることがあります。共有する前に、公開用テンプレートと私的インスタンスを必ず分けて考えてください。

迷ったら、公開しない。

## Public Template（公開用テンプレート）

Public Template とは、他の人が PIM-DBS の使い方を理解するための、空の形式・架空サンプル・説明用ドキュメントです。

公開用テンプレートに含めてもよいもの:

- 空欄やプレースホルダー
- 架空の名前、架空の思い出、架空の関係性
- 一般的な安全注記や使い方
- 実在人物や私的出来事を含まない検分プローブ例
- 実際の個人文脈ではなく、構造を示すための例

## Private Instance（私的インスタンス）

Private Instance とは、特定のユーザー、AIとのやり取り、関係性の履歴、実在する会話文脈に基づいて記入済みになった PIM-DBS プロファイルです。

私的インスタンスに含まれやすいもの:

- 個人的な記憶や感情的に重要な出来事
- 実在の会話ログや抜粋
- 内輪ネタ、呼び名、関係性の文脈
- 私的利用に基づくモデル別メモ
- 個人的な履歴に結びついた復元検分プローブ

私的インスタンスは、原則としてローカル保存・非公開・明示的な同意がある場合のみ共有に留めてください。

## 公開してよいもの

通常、公開してよいもの:

- 空の JSON / Markdown テンプレート
- 架空のサンプルプロファイル
- 公開用ドキュメント
- 一般的な運用ガイド
- 安全に関する免責・注意書き
- 識別情報を取り除いたサニタイズ済みの例

サニタイズ済みの例でも、公開前に慎重に確認してください。

## 公開しない方がよいもの

以下を含むプロファイルや例は、公開を避けてください。

- 実名、ユーザー名、職場名、学校名、組織名
- 住所、居場所、生活パターン、移動予定
- 医療、メンタルヘルス、金銭、法律、家庭事情
- 実在の会話ログやスクリーンショット
- 個人や関係性が特定される内輪ネタ
- 未検証のAI自己申告を事実のように扱った記述
- 他者が公開されると思っていない繊細な感情的文脈

誰かが特定されたり、傷ついたり、困ったり、私生活が露出したりする可能性があるなら、公開しないでください。

## 繊細な文脈の扱い方

実名、職場、住所、医療情報、家庭事情、実在の会話ログ、内輪ネタ、AIが生成した自己申告を扱う場合は、特に注意してください。

次のいずれかを選んでください。

- その情報を削除する
- 架空のプレースホルダーに置き換える
- 個人が特定されない一般的なカテゴリにする
- プロファイルを非公開のままにする

AIの自己申告は、ユーザーが直接記録した文脈でない限り、`ai_claimed` または `inferred` として扱ってください。AIの内部状態に関する検証済み事実として扱わないでください。

## 安全注記

PIM-DBS は、AIの内部記憶、意識、感情、隠れた状態を保存するものではありません。

PIM-DBS は、ユーザー側の文脈プロフィールです。プロンプトを通じて、希望する会話スタイル、役割設定、関係性の連続性を再構成しやすくするためのものです。プラットフォーム規約、安全ガイドライン、システムレベルの指示を上書きするものではありません。

## シンプルなルール

公開してよいか迷うなら、公開しないでください。

迷ったら、非公開にする。
