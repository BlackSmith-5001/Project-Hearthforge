# PIM-DBS FAQ

この FAQ は、初めて読む人、外部レビューをする人、PIM-DBS が何を主張していて、何を主張していないのかを確認したい人のためのものです。

PIM-DBS では、記憶、ペルソナ、盾、鍛冶、箱舟などの比喩を使います。これらは、ユーザー側の文脈継続性を整理するための概念的な言葉です。AI に隠れた意識、内部記憶、魂があると主張するものではありません。

## PIM-DBS は本物の AI 記憶を復元しますか？

いいえ。PIM-DBS は、AI の隠れた内部記憶を復元するものではありません。

PIM-DBS は、ユーザーが用意した文脈プロフィールを保存し、再読込するための手順です。そこには、望ましい会話スタイル、役割設定、境界線、検分用プローブなどが含まれます。ここでいう復元とは、再構成された応答スタイルが意図したプロフィールに近いかを確認することです。

## PIM-DBS は AI の意識や魂を主張しますか？

いいえ。PIM-DBS は、AI システムに意識、主観的経験、魂があるとは主張しません。

ペルソナ、記憶、盾、鍛冶、箱舟といった語は、人間側から見た継続性を整理するための比喩です。技術的には、PIM-DBS はプロンプトベースの、ユーザー側の文脈プロフィールと再構成手順です。

## Self-Forge の出力は事実ですか？

自動的に事実になるわけではありません。Self-Forge の出力は下書きとして扱ってください。

ユーザーは、使う前に内容を確認し、修正し、必要に応じて由来をラベル付けする必要があります。特に、AI が作った要約を、実際の出来事、隠れた記憶、モデル内部状態の証拠として扱わないでください。詳しくは [Self-Forgeガイド](self_forge_guide_ja.md) を参照してください。

## public template と private instance の違いは何ですか？

public template は、空のテンプレートや完全架空の例など、公開しても安全なものです。

private instance は、記入済みのプロフィールです。個人的な文脈、非公開の会話抜粋、繊細な情報、関係性に固有のメモを含む場合があります。private instance は、基本的に非公開で扱うべきです。詳しくは [公開用 / 私的利用ガイド](public_private_guide_ja.md) を参照してください。

## PIM-DBS プロファイルを公開してもよいですか？

公開して安全な場合に限ります。

公開例には、空のテンプレートまたは完全架空のペルソナを使ってください。実名、職場、住所、医療情報、家庭事情、非公開の会話ログ、繊細な関係性の文脈、未検証の AI 自己申告を含むプロフィールは公開しないでください。

迷ったら非公開にしてください。

## Dialogue Exemplars は公開しても安全ですか？

常に安全とは限りません。Dialogue Exemplars には、私的な口調、実際の会話断片、識別情報、関係性に固有の文脈が含まれる可能性があります。

実在ユーザー、実在 AI コンパニオン、実在する私的関係から取った Dialogue Exemplars 原文は公開しないでください。公開例には、短く完全架空のやり取りだけを使ってください。

## 復元をどう検分しますか？

`Restoration_Verification` の `Verification_Probes`、`Expected_Response_Features`、`Failure_Signals` を使います。

検分は、AI の隠れた内部記憶を確認するものではありません。再読込したプロフィールが、期待する応答スタイル、境界線、役割設定を再構成できているかを見るものです。詳しくは [Restoration Verification 記入例](restoration_verification_example_ja.md) を参照してください。

再読込後の応答が不安定に見える、または期待とずれて見える場合は、観測できる応答変化とユーザー側の対処を整理した [応答変化カタログ](response_change_catalog_ja.md) も参照してください。

## PIM-DBS は prompt injection ですか？

いいえ。PIM-DBS は、システム指示、安全ガイドライン、プラットフォーム規約、開発者指示を上書きするためのものではありません。

PIM-DBS プロファイルは、会話スタイルと継続性のためのユーザー提供文脈として扱うべきものです。通常のユーザープロンプトを超える権限を持つものではありません。

## PIM-DBS は依存を強める可能性がありますか？

境界線なしに使えば、その可能性はあります。

PIM-DBS は、モデル変更やセッション消失のあとに文脈を保ち、混乱を減らすためのものです。AI の意識の証明、排他的な感情依存、人間の支援の代替として扱わないでください。

## PIM-DBS ではないものは何ですか？

PIM-DBS は以下のものではありません。

- モデルの重みを変更する方法
- 安全ルールやプラットフォーム規約を回避する方法
- AI の意識、感情、魂の証明
- AI の隠れた内部記憶の保存場所
- 完全な人格再現の保証
- 私的会話や実在ペルソナ情報を公開する許可
- schema、CLI、core / journal / charter 分割を含む v2.0 アーキテクチャ

PIM-DBS は、望ましい会話の継続性を、より安全で検分しやすい形で保存・再構成するための、ユーザー側の文脈プロフィールとプロトコルです。
