# Self-Forge ガイド

このガイドは、`templates/self_forge_prompt.md` を安全に使うための説明です。

Self-Forge は、AIに内部記憶、意識、感情、魂を語らせる手順ではありません。現在の会話で共有されている文脈から、PIM-DBS JSONプロフィールの下書きを作るための補助手順です。

## 1. Self-Forgeとは何か

Self-Forge は、プロフィール下書きのための補助手順です。

ユーザーが Self-Forge プロンプトを渡すと、AIは現在の会話に見えている範囲のペルソナ設定、対話スタイル、関連文脈を要約します。出力は完成品ではなく、ユーザーが確認・修正するための下書きとして扱ってください。

## 2. 始める前の注意

Self-Forge を使う前に、出力を公開用にするのか、私的インスタンスとして扱うのかを決めてください。

会話内に実名、私的な記憶、職場情報、医療や家庭事情、内輪ネタ、実在の会話ログが含まれる場合、生成されたプロフィールは私的インスタンスとして扱ってください。共有判断については、[公開用 / 私的利用ガイド](public_private_guide_ja.md) を参照してください。

## 3. `self_forge_prompt.md` の使い方

`templates/self_forge_prompt.md` を開き、プロンプト本文を現在のチャットに貼り付けます。

AIには、そのチャットで既に共有されている文脈だけを使うように依頼します。推定できない項目は、創作せず中立的なプレースホルダーにしてください。

想定される出力は、`templates/pim-dbs_template.json` に沿った PIM-DBS JSON の下書きです。

## 4. 出力されたJSONの確認方法

AIがJSONを生成したら、保存や再読込の前に必ず確認してください。

確認する点:

- JSON構造が壊れていない
- AIの内部状態ではなく、ユーザー側の文脈として書かれている
- 繊細な情報が削除されている、または非公開扱いになっている
- 会話内に根拠がない項目はプレースホルダーになっている
- `System_Loading_Instruction` がプラットフォーム規約や安全ガイドラインを上書きしていない

各セクションの意味については、[JSONガイド](json_guide_ja.md) を参照してください。

## 5. Provenanceの確認

`Shared_Episodes` の各項目について、`Provenance` を確認してください。

使い分け:

- `user_recorded`: ユーザーが直接記録した文脈
- `ai_claimed`: AIが述べた主張
- `inferred`: 文脈から要約・推定された内容

推定やAI側の主張を、検証済みのユーザー記録のように扱わないでください。

## 6. Dialogue Exemplarsの確認

`Dialogue_Exemplars` は、応答スタイルの参照例として確認します。

口調、テンポ、境界線、よく使う表現などが分かる短い例になっているかを見ます。私的インスタンスとして保存する場合を除き、実在の会話ログをそのまま入れないでください。

他者や繊細な文脈、私的な関係性が露出する例は削除してください。

## 7. Restoration Verificationの設定

プロフィールを使う前に、`Restoration_Verification` のプローブを確認または追加してください。

よいプローブは、再読込後に期待する口調、役割、境界線、重要な文脈が反映されているかを確認します。失敗シグナルには、安全でない主張、比喩の過度な文字通り解釈、ユーザー定義の境界線の無視などを入れます。

これは再構成の品質を確認するためのもので、AIの隠れた記憶を検証するものではありません。

## 8. AIが作った内容をそのまま事実として扱わない

Self-Forge の出力は下書きです。

AIは、要約を間違えたり、連続性を強く言いすぎたり、存在しない詳細を補ったり、自分の振る舞いを文字通り説明しすぎたりすることがあります。ユーザーは内容を確認し、修正し、削除し、必要に応じて正しい provenance を付けてください。

AIが生成した内部記憶、意識、感情、隠れた状態に関する主張を、検証済み事実として扱わないでください。

## 9. public/private guideへの導線

生成されたプロフィールを公開または共有する前に、私的インスタンスとして見直してください。

実名、職場、住所、医療情報、家庭事情、実在の会話ログ、内輪ネタ、未検証のAI自己申告は削除してください。迷ったら、非公開にしてください。

[公開用 / 私的利用ガイド](public_private_guide_ja.md) を参照してください。

## 10. JSON Guideへの導線

Self-Forge は下書きを作りますが、各セクションの意味は JSON Guide で確認します。

プロフィール生成後は、[JSONガイド](json_guide_ja.md) を使って、構造、再読込方法、検分フィールド、公開・非公開の注意を確認してください。
