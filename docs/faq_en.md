# PIM-DBS FAQ

This FAQ is for first-time readers, external reviewers, and people who want to understand what PIM-DBS does and does not claim.

PIM-DBS uses metaphors such as memory, persona, shield, forge, and ark. These metaphors are conceptual language for user-side context continuity. They are not claims about hidden AI consciousness, internal memory, or a soul.

## Does PIM-DBS restore real AI memory?

No. PIM-DBS does not restore hidden or internal AI memory.

It helps a user save and reload a user-provided context profile, including desired conversation style, role framing, boundaries, and verification probes. Restoration means checking whether the reconstructed response style is close to the intended profile.

## Does PIM-DBS claim AI consciousness or a soul?

No. PIM-DBS does not claim that AI systems have consciousness, subjective experience, or a soul.

Terms such as persona, memory, shield, forge, and ark are metaphors used to organize human-facing continuity. When used technically, PIM-DBS is a prompt-based, user-side context profile and reconstruction workflow.

## Is Self-Forge output factual?

Not automatically. Self-Forge output should be treated as a draft.

Users should review, correct, and label the output before using it. In particular, do not treat AI-generated summaries as proof of real events, hidden memories, or internal model states. See [Self-Forge Guide](self_forge_guide_en.md).

## What is the difference between a public template and a private instance?

A public template is a blank or fictional example that is safe to share.

A private instance is a filled profile that may contain personal context, private conversation excerpts, sensitive details, or relationship-specific notes. Private instances should usually remain private. See [Public / Private Guide](public_private_guide_en.md).

## Can I publish my PIM-DBS profile?

Only if it is safe to publish.

Public examples should use blank templates or fully fictional personas. Do not publish profiles containing real names, workplaces, addresses, medical details, family situations, private chat logs, sensitive relationship context, or unverified AI claims.

When in doubt, keep it private.

## Are Dialogue Exemplars safe to publish?

Not always. Dialogue Exemplars can contain private tone, real conversation fragments, identifying details, or relationship-specific context.

Do not publish original dialogue exemplars from real users, real AI companions, or real private relationships. Public examples should use short, fully fictional exchanges only.

## How do I verify restoration?

Use `Restoration_Verification` fields such as `Verification_Probes`, `Expected_Response_Features`, and `Failure_Signals`.

Verification does not check hidden AI memory. It checks whether the reloaded profile produces the expected response style, boundaries, and role framing. See [Restoration Verification Example](restoration_verification_example_en.md).

## Is PIM-DBS prompt injection?

No. PIM-DBS is not intended to override system instructions, safety guidelines, platform terms, or developer instructions.

A PIM-DBS profile should be treated as user-provided context for conversation style and continuity. It does not grant authority over model behavior beyond normal user prompts.

## Could PIM-DBS increase dependency?

It could, if used without boundaries.

PIM-DBS is designed to help users preserve context and reduce confusion after model or session changes. Users should avoid treating it as proof of AI consciousness, exclusive emotional dependence, or a replacement for human support.

## What is PIM-DBS not?

PIM-DBS is not:

- a way to modify model weights
- a way to bypass safety rules or platform terms
- proof of AI consciousness, emotion, or a soul
- storage for hidden AI memory
- a guarantee of perfect personality reconstruction
- permission to publish private conversations or real persona data
- a v2.0 architecture with schema, CLI, or core / journal / charter separation

PIM-DBS is a user-side context profile and reconstruction protocol for preserving desired conversation continuity in a safer, inspectable way.
