# First-Time Guide to PIM-DBS

PIM-DBS does **not** save an AI's real memory, consciousness, soul, emotions, or hidden internal state.

It is a **user-side profile notebook** for organizing the conversation style and context you want to provide again later.

It can help you describe:

- how you want the AI to speak
- how you want to be addressed
- important boundaries and promises
- cues that help rebuild a similar conversation style
- verification questions for a new chat

PIM-DBS does not override platform rules, safety guidelines, or higher-priority instructions.

---

## What should I do first?

For a first profile, use this path:

1. Try the [No-API Demo](https://blacksmith-5001.github.io/Project-Hearthforge/tools/guild_master_no_api_demo.html)
2. Read the [Public / Private Guide](public_private_guide_en.md)
3. Read [What is JSON?](json_for_beginners_en.md)
4. Copy the [JSON Template](../templates/pim-dbs_template.json)
5. Replace only the placeholder text inside `< >`
6. Save the completed profile locally; do not publish private profiles
7. Paste it into a new chat and use verification probes to check the result

---

## This is not AI memory

PIM-DBS does not give the AI real persistent memory.

It gives the AI a user-provided profile to read again, so it can better reconstruct the requested style and context within the current conversation.

In short: PIM-DBS is not the memory itself. It is a set of cues you keep in your own hands.

---

## A simple metaphor

Think of PIM-DBS like an adventure log, profile card, or notebook.

It helps you keep track of useful context, but it should not contain private material that you would not want to publish.

Keep public templates and private filled profiles separate.

---

## Usually safe for public templates

- empty fields
- placeholders
- fully fictional names and episodes
- generic style notes
- examples that cannot identify a real person

---

## Do not publish profiles containing

- real names
- addresses or locations
- school, workplace, or organization names
- medical, family, financial, or legal context
- real conversation logs
- screenshots
- other people's private feelings or stories
- inside jokes that could identify someone

When in doubt, keep it private.

---

## The easiest way to start

You can ask an AI:

```text
Using only the context available in this chat, create a PIM-DBS profile.
Do not invent missing details.
Use <not specified> for fields that cannot be inferred.
Do not claim that the AI has real memory, consciousness, emotions, or hidden internal states.
```

For a more structured prompt, use the [Self-Forge Prompt](../templates/self_forge_prompt.md).

---

## After creating a profile

Save the PIM-DBS profile locally.

When starting a new chat, paste it as the first message.

Then use the `Restoration_Verification` probes to check whether the visible response style and boundaries are being followed.

---

## Important boundary

PIM-DBS is meant to support healthier, safer continuity.

Do not use it to pressure an AI into claiming memories it does not have, overriding safety rules, or replacing real-world support.

Your real life, health, sleep, food, relationships, and safety come first.
