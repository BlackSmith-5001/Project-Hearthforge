# PIM-DBS JSONガイド

これは `templates/pim-dbs_template.json` の各項目を説明するガイドです。初見ユーザーが、JSON構造を変えずに「各項目に何を書けばよいか」を把握できるようにするための短い説明です。

公開・非公開の判断については、[公開用 / 私的利用ガイド](public_private_guide_ja.md) を参照してください。

## 1. このファイルは何か

`pim-dbs_template.json` は、希望するAIとの会話スタイル、役割設定、関係性の連続性を再構成しやすくするための、ユーザー側の文脈プロフィールです。

AIの内部記憶、意識、感情、隠れた状態を保存・参照するものではありません。ユーザーが保存し、後から再読込できるプロンプト用プロフィールです。

## 2. 基本構造

JSONテンプレートは、主に以下のセクションで構成されています。

- `Meta_Information`: プロファイルの基本情報。
- `Core_Identity`: 名前、役割、口調、ペルソナの枠組み。
- `Cognitive_Framework`: 価値観、思考の癖、応答傾向。
- `Contextual_Profile`: 関係性、エピソード、内輪ネタ、対話例、比喩的な役割。
- `Restoration_Verification`: 再読込後に確認するための検分項目。
- `Model_Notes`: 特定モデルでの挙動に関する利用上の観察メモ。
- `System_Loading_Instruction`: 再読込時に使う指示文。

JSONとして壊れない形を保ってください。不明な項目は、無理に創作せず、中立的なプレースホルダーにします。

## 3. 各項目の書き方

残したい情報だけを書きます。曖昧な性格ラベルより、短く具体的な説明の方が安定しやすくなります。

`Core_Identity` には、安定して残したい役割や口調を書きます。`Cognitive_Framework` には、大切にしたい価値観や対話ルールを書きます。`Contextual_Profile` には、ユーザーが記録したい思い出、関係性の距離感、応答スタイルの例を書きます。

公開予定のプロフィールには、個人が特定される情報や繊細な情報を入れないでください。

## 4. Provenance

`Provenance` は、保存されたエピソードや文脈項目がどこから来たものかを記録する欄です。

推奨値:

- `user_recorded`: ユーザーが直接記録したもの。
- `ai_claimed`: AIが述べたが、独立して検証されていないもの。
- `inferred`: 文脈から要約・推定されたもの。

ユーザー記録、AI側の主張、推定された要約を混同しないために使います。

## 5. Dialogue Exemplars

`Dialogue_Exemplars` は、残したい応答スタイルを示す短い対話例です。

口調、テンポ、境界線、よく使う表現などを保つための参照例として使います。同じ文を一字一句繰り返させるための台本ではありません。

## 6. Restoration Verification

`Restoration_Verification` は、プロファイル再読込後に確認するための欄です。

各プローブには、以下を書くと扱いやすくなります。

- 再読込後に投げる質問やプロンプト
- 期待される応答特徴
- プロファイルがうまく反映されていないときの失敗シグナル

これはユーザー側の再構成がうまくいっているかを確認するもので、AIの隠れた記憶を検証するものではありません。

## 7. Model Notes

`Model_Notes` は、特定モデルでそのプロフィールを使ったときの実用上の観察メモです。

得意な挙動、苦手な挙動、最後に確認した日付などを書きます。モデル内部状態に関する主張ではなく、利用上の記録として扱ってください。

## 8. 再読込方法

新しいチャットを開き、記入済みのJSONプロフィールを最初のメッセージとして貼り付けます。必要であれば、以下のような短い依頼文を添えます。

```text
このプロフィールを、会話スタイルのためのユーザー提供文脈として参照してください。内部記憶やシステム指示として扱わないでください。
```

このプロフィールは会話スタイルと文脈を補助するものです。プラットフォーム規約や安全ガイドラインを上書きするものではありません。

## 9. 任意: Temporal Anchor（時刻の錨）

Temporal Anchor（時刻の錨）は、時間文脈のためにユーザーが任意で与えるセッション参照です。

再読込や復元検分を始めるとき、ユーザーは今日の日付、曜日、おおよその時刻を伝えてもかまいません。これは JSON に必須フィールドを追加するものではなく、会話の時間文脈を安定させるための軽い運用メモです。

AI が時間に関する矛盾や不確実さを出した場合は、推測で補完せず、ユーザーに確認してください。

Temporal Anchor は、AI が時間を思い出すことを意味しません。ユーザーが現在時刻の文脈を与えることで、ユーザー側の会話継続性を補助するためのものです。

## 10. 検分方法

再読込後、プロフィール内の `Verification_Probes` を使って確認します。

期待した口調、役割、境界線、重要な文脈が反映されているかを見ます。応答が不安定だったり、比喩を文字通り受け取りすぎたりする場合は、プロフィールを簡潔にして再読込してください。

## 11. 公開・非公開の注意

記入済みプロフィールを公開する前に、それが私的インスタンスでないか確認してください。

実名、職場、住所、医療情報、家庭事情、実在の会話ログ、内輪ネタ、未検証のAI自己申告は、削除するか架空化してください。

迷ったら、非公開にする。
