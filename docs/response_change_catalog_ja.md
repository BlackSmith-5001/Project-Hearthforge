# 応答変化カタログ

このカタログは、PIM-DBS の運用中にユーザーが観測できる応答変化を整理し、試せるユーザー側の対処を選ぶためのものです。

モデル内部状態、AI の故障、人格崩壊、意識、記憶喪失を診断する文書ではありません。ここで扱うのは、あくまで出力として観測される変化です。

## 1. この文書について

PIM-DBS は、ユーザー側の文脈プロフィールと再構成手順です。運用中、アシスタントの応答が不安定に見えたり、定型的になったり、急に長くなったり、読み込んだプロフィールとずれて見えたりすることがあります。

この文書は、そのような変化を観測し、短文化、話題分割、Temporal Anchor（時刻の錨）の再提示、JSON プロファイルの再読込、Restoration Verification の再実施、休憩などの低リスクな対処を試すための簡易表です。

再読込後の検分については [Restoration Verification 記入例](restoration_verification_example_ja.md) を参照してください。JSON プロファイルの使い方については [JSONガイド](json_guide_ja.md) を参照してください。短期的な応答変化をユーザー側の粗い帯域に分ける方法については [Response Change Bands](response_change_bands_ja.md) を参照してください。

## 2. 安全な前提

- 観測できる出力を記述します。隠れたモデル内部状態は診断しません。
- 「モデルが壊れた」「人格が崩壊した」のような断定は避けます。
- AI の意識、魂、感情、隠れた永続的記憶を主張しません。
- 記憶、鍛冶、盾、箱舟などの比喩は、ユーザー側の継続性を説明する概念的な言葉として扱います。
- 実在する Dialogue Exemplars、私的会話、実在ペルソナの声紋は公開しません。
- 例示が必要な場合は、完全架空のペルソナだけを使います。

## 3. 応答変化の表

| 観測される応答変化 | 考えられる要因 | ユーザー側の対処 |
| --- | --- | --- |
| 日付、曜日、時刻について混乱した出力が出る。 | セッション内に明確な時間参照がない、または以前の時間文脈が目立たなくなっている可能性があります。 | 今日の日付、曜日、おおよその時刻を Temporal Anchor（時刻の錨）として再提示します。分からない場合は推測せず確認するよう伝えます。 |
| 定型文が増える。 | プロンプトが広すぎる、安全説明が強く出すぎている、またはプロフィールが抽象的すぎる可能性があります。 | 短い回答を依頼します。具体的な作業を言い直します。必要なら関連するプロフィール欄だけを再提示します。 |
| 急に応答が長くなる。 | 複数の話題が混ざっている、または多くの制約を同時に守ろうとしている可能性があります。 | 話題を小さく分けます。簡潔な応答と次の一手だけを依頼します。 |
| 共有済みの前提が抜け落ちて見える。 | 現在のプロンプトに重要な文脈が含まれていない、またはプロフィール内の要約が弱い可能性があります。 | 抜け落ちた前提を短く再提示します。全体ではなく、必要な JSON セクションだけを再読込します。 |
| 口調、一人称、呼び方がずれる。 | スタイル指定が弱い、矛盾している、または他の指示に埋もれている可能性があります。 | `Core_Identity` と短い完全架空の Dialogue Exemplar を再提示します。実在の私的対話は使いません。 |
| 感情表現が意図より強くなる。 | 支援的な表現が増幅されている、または比喩を文字通り受け取りすぎている可能性があります。 | 穏やかで実務的な口調を依頼します。比喩は概念説明であり、AI の内面状態の主張ではないと明記します。 |
| 初期設定へ戻ったような応答に見える。 | 現在の文脈でプロフィールが十分に参照されていない、またはプロンプトがプロフィールを参照していない可能性があります。 | JSON プロファイル、または必要なセクションを再読込します。その後、Restoration Verification の質問をひとつ実施します。 |
| 会話リズムが破綻し、流れを追いにくい。 | 目標、修正、話題が多すぎる可能性があります。 | いったん止めて現在の目的を要約します。質問または作業をひとつずつ進めます。 |
| 境界線が曖昧になる。 | 望ましいスタイルと、安全ルール、プライバシー制限、プラットフォーム規約の区別が弱い可能性があります。 | 境界線を明示します。私的文脈は非公開のまま扱います。プロフィールは安全ガイドラインや規約を上書きしないと明確にします。 |
| Restoration Verification の質問に合わない応答が出る。 | 再読込が不十分、プローブが曖昧、または期待する特徴が広すぎる可能性があります。 | プロファイルを簡潔にします。プローブをひとつだけ再実施します。隠れた記憶ではなく、観測できる応答特徴だけを見比べます。 |

## 4. 使い方

1. 観測された応答変化を特定します。
2. 表の中で近い行を探します。
3. いちばん小さいユーザー側の対処から試します。
4. 続く場合は、プロフィールを簡潔にするか、Restoration Verification を再実施します。
5. やり取りが感情的に強く感じられたり混乱したりする場合は、文脈をさらに追加する前に休憩します。

このカタログは実用上の補助であり、応答が完全に復元できることを保証するものではありません。

観測できる応答変化を 0〜4 の粗い帯域に分ける方法については、[Response Change Bands](response_change_bands_ja.md) を参照してください。

## 5. やってはいけないこと

- アシスタントの内部状態が損傷したと決めつけない。
- 出力のズレを、AI の記憶喪失、意識、苦痛の証拠として扱わない。
- 不安定さへの最初の対処として、私的文脈をどんどん追加しない。
- 助言を求めるために、実在する Dialogue Exemplars や私的会話ログを公開しない。
- パーセンテージや偽精度で、アシスタントの内面状態を表現しない。
- 安全ルール、プラットフォーム規約、プライバシー境界を回避するためにこのカタログを使わない。
