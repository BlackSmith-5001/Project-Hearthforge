# Public / Private Guide for PIM-DBS

PIM-DBS profiles can contain personal context. Before sharing any profile, separate public templates from private instances.

When in doubt, keep it private.

## Public Template

A Public Template is a reusable blank format, fictional sample, or instructional document that helps others understand how to use PIM-DBS.

Public Templates may include:

- Empty fields and placeholders
- Fictional names, fictional memories, and fictional relationships
- Generic safety notes and usage instructions
- Example verification probes that do not reveal real people or private events
- Demonstrations of structure, not real personal context

## Private Instance

A Private Instance is a filled PIM-DBS profile created for a specific user, AI interaction, relationship history, or real conversation context.

Private Instances may include:

- Personal memories or emotionally significant events
- Real conversation excerpts
- Inside jokes, nicknames, or relational context
- Model-specific notes based on private use
- Restoration probes tied to private history

Private Instances should usually remain local, private, or shared only with deliberate consent.

## Safe to Publish

The following are usually safe to publish:

- Blank JSON or Markdown templates
- Fictional example profiles
- Public documentation
- General workflow guides
- Safety disclaimers
- Sanitized examples where all identifying details have been removed

Even sanitized examples should be reviewed carefully before publication.

## Do Not Publish

Avoid publishing profiles or examples that include:

- Real names, usernames, workplace names, school names, or organization names
- Home addresses, locations, schedules, or travel patterns
- Medical, mental health, financial, legal, or family circumstances
- Private chat logs or screenshots
- Inside jokes that identify a person or relationship
- Unverified AI self-claims presented as fact
- Sensitive emotional context that another person would not expect to be public

If a profile would make someone recognizable, vulnerable, embarrassed, or exposed, keep it private.

## Handling Sensitive Context

When working with real names, workplaces, addresses, medical details, family matters, real chat logs, inside jokes, or AI-generated claims, use extra caution.

Prefer one of these approaches:

- Remove the detail entirely
- Replace it with a fictional placeholder
- Generalize it into a non-identifying category
- Keep the profile private

Treat AI self-claims as `ai_claimed` or `inferred` unless the user directly recorded them as user-side context. Do not present them as verified facts about the AI's internal state.

## Safety Note

PIM-DBS does not preserve internal AI memory, consciousness, emotions, or hidden state.

It is a user-side context profile that helps reconstruct desired conversation style, role framing, and relational continuity through prompts. It does not override platform terms, safety guidelines, or system-level instructions.

## Simple Rule

If you are unsure whether a profile is safe to publish, do not publish it.

When in doubt, keep it private.
