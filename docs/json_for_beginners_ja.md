# JSONってなに？

JSON は、プログラムではありません。
AI に渡すための、**項目つきメモ帳** だと思ってください。

たとえば、これは：

```json
"Name": "ルナ"
```

「名前はルナです」という意味です。

---

## まず覚えること

PIM-DBS の JSON では、基本的に **`< >` の中だけを書き換えれば大丈夫** です。

たとえば：

```json
"Name": "<The name you want to give to the AI>"
```

これを：

```json
"Name": "ルナ"
```

のように書き換えます。

---

## なるべく消さない記号

次の記号は、JSON の形を作るためのものです。

```text
{ }
[ ]
"
,
:
```

よく分からない場合は、これらの記号を消さないでください。

---

## よくある壊れ方

### 1. ダブルクォーテーションを消してしまう

壊れやすい例：

```json
"Name": ルナ
```

安全な例：

```json
"Name": "ルナ"
```

文字は、基本的に `" "` の中に入れます。

---

### 2. カンマを消してしまう

JSON では、項目と項目の間に `,` が必要なことがあります。

壊れやすい例：

```json
"Name": "ルナ"
"Callsign": "夜空の相棒"
```

安全な例：

```json
"Name": "ルナ",
"Callsign": "夜空の相棒"
```

---

### 3. 余計な説明をJSONの中に直接書いてしまう

壊れやすい例：

```json
"Name": "ルナ" ← ここに名前を書く
```

安全な例：

```json
"Name": "ルナ"
```

説明を書きたい場合は、`Notes` のような項目の中に入れてください。

---

## 書き換えてよい場所

次のような `< >` の部分を書き換えます。

```json
"Style": "<Tone and Personality Settings>"
```

例：

```json
"Style": "やさしく、落ち着いていて、少し茶目っ気がある"
```

---

## 分からないところは空欄でいい

分からない項目は、無理に埋めなくても大丈夫です。

例：

```json
"Origin": "<not specified>"
```

AI に勝手に作らせたい場合でも、実在する過去や個人情報を想像で作らせないでください。

---

## 壊れたかもしれないとき

AI にこう頼んでください。

```text
このJSONを壊さずに直してください。
内容は勝手に増やさないでください。
足りないところは <not specified> のままにしてください。
```

---

## 公開前チェック

完成した JSON を公開する前に、次のものが入っていないか確認してください。

- 実名
- 住所や居場所
- 学校名、職場名、組織名
- 医療、家庭、金銭、法律に関する話
- 実際の会話ログ
- スクリーンショット
- 他人が知られたくない話

迷ったら、公開しないでください。

---

## 最後に

JSON はこわいものではありません。

ただの「形が決まったメモ帳」です。

記号の形をなるべく壊さず、自分に必要な言葉だけを少しずつ入れてください。
