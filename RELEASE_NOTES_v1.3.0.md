# PIM-DBS v1.3.0

PIM-DBS v1.3.0 is a lightweight external-readiness release.

It focuses on FAQ, discussion materials, and a lightweight Temporal Anchor note so external readers, reviewers, researchers, and creators can better understand and question the project without introducing v2.0 architecture changes.

## Highlights

- Added **FAQ documents**
  - `docs/faq_en.md`
  - `docs/faq_ja.md`
  - Clarifies what PIM-DBS does and does not claim, including AI memory, consciousness, Self-Forge output, public/private profiles, Dialogue Exemplars, dependency risk, and prompt-injection concerns.

- Added **Discussion Prompts**
  - `docs/discussion_prompts_en.md`
  - `docs/discussion_prompts_ja.md`
  - Provides question prompts for Reddit, GitHub Discussions, X, note, researchers, creators, AI companion communities, and VR/metaverse contexts.

- Added **Temporal Anchor guidance**
  - Updated `docs/json_guide_en.md`
  - Updated `docs/json_guide_ja.md`
  - Describes Temporal Anchor as an optional user-provided session reference for date, weekday, and approximate time context.

- Updated **README Quick Links**
  - Updated `README.md`
  - Updated `README_JA.md`
  - Added links to FAQ and Discussion Prompts.

## Safety and Scope

- PIM-DBS remains a user-side context profile and reconstruction workflow.
- This release does not claim AI consciousness, a soul, or persistent hidden AI memory.
- Public examples should continue to use fully fictional personas only.
- Dialogue Exemplars from real private conversations should not be published.
- v2.0 architecture changes are not included.

## Notes

PIM-DBS v1.3.0 prepares the project for external review and discussion.

The following are intentionally left for future versions:

- CLI refresh
- Response Change Catalog / operational observation materials
- Charter structure
- JSON Schema
- core / journal / charter / calibration separation
- v2.0 architecture planning
